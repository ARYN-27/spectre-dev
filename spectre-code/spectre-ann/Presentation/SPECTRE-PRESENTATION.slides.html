  • Define the number of rows and columns of the images
  • Split the input array of images into training and test sets with the defined percentage ( 75% training 25% testing)
  • Extract the images for the training set and testing set
  • Reshape the images in training and testing sets to have specific dimension (img_row, img_col, 3)

Creating new model based on vgg19¶

  • Modifying a pre-existing model, "model_vgg19_conv," to create a new model, "my_model."
  • The code first assigns the output of the pre-existing model, "model_vgg19_conv," to the variable "x"; Then it flattens the output using "x = layers.Flatten()(x)".
  • It then uses a dense layer of 128 neurons and uses the "relu" activation function to process the output. This is assigned to the same variable "x".
  • The next line adds another dense layer with 1 neuron and using the activation function "sigmoid". It assigns the same variable "x" and giving it a name 'predictions'
  • Then the code creates a new model, "my_model," using the inputs from the pre-existing model and the outputs from the new layers added to the variable "x".
  • The last line "my_model.summary()" prints a summary of the new model created.

In simple terms, the code is modifying the pre-existing model to classify images based on their contents by adding layers to it and creating a new model to do so.

Configuring the new model¶

  • The first line, "my_model.compile," sets up the training process for the model by defining the loss function, optimization algorithm, and metrics to be used.

    • The "loss" parameter is set to "keras.losses.binary_crossentropy" which is a loss function used for binary classification problems, where the output can be either 0 or 1.
    • The "optimizer" parameter is set to "keras.optimizers.RMSprop()" which is an optimization algorithm used to update the model's weights during training.
    • The "metrics" parameter is set to "accuracy" which is used to evaluate the model's performance during training.

  • The next two lines set the batch size and number of epochs for the training process.

    • The "batch_size" variable is set to 32, meaning that the model will process 32 images at a time before updating its weights.
    • The "epochs" variable is set to 2, meaning that the model will be trained on the entire dataset 2 times.

In simple terms, this code is setting up the training process for the model by defining how the model should be optimized during training and how the performance will be evaluated. It also sets the number of images that the model should process before updating its weights and the number of times that the model should see the entire data set.

Training the model¶

The "my_model.fit" function trains the model on the input data set, "X_train," and the corresponding labels, "y_train."

It takes several parameters to configure the training process:

  • The "batch_size" parameter is set to the value of the variable "batch_size" which is set at 32. The model will process 32 images at a time before updating its weights.
  • The "epochs" parameter is set to the value of the variable "epochs" which was define as 2. This means that the model will be trained on the entire dataset 2 times.
  • The "verbose" parameter is set to 1, which means that the training progress will be displayed on the console while the model is being trained.
  • The "validation_split" parameter is set to 0.2, which means that 20% of the training data will be used as validation data to evaluate the model's performance during training.
